# Section Definitions

This file defines the reference categories for the Copilot SDK skill.

## 1. Critical Verification (critical)

**Impact:** CRITICAL

Version-sensitive SDK facts, public-preview caveats, docs lookup paths, and source confirmation.

## 2. Critical Safety (safety)

**Impact:** CRITICAL

Permissions, auth, BYOK, user data, external tools, shell/file/network access, and failure behavior.

## 3. High Architecture (architecture)

**Impact:** HIGH

Session design, tools, MCP, custom agents, skills, persistence, events, and source-of-truth ownership.

## 4. High Implementation (implementation)

**Impact:** HIGH

Language-specific API patterns, Go defaults, examples, lifecycle cleanup, and code generation rules.

## 5. Medium Prompt Quality (prompt)

**Impact:** MEDIUM

Concise directives, reusable prompt modules, anti-shallow planning checks, and eval prompts.

## 6. Medium Validation (validation)

**Impact:** MEDIUM

Risk-based tests, manual checks, production readiness, and evaluation workflow.
