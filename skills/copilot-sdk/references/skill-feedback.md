# Skill Feedback

Use this when the user says the `copilot-sdk` skill gave incorrect guidance, missed important information, or feels too generic.

## Steps

1. Identify the failing behavior.
2. Identify the specific file and section that caused or failed to prevent the issue.
3. Compare against a stronger external skill pattern when useful.
4. Patch the skill instructions or references.
5. Validate the skill package.
6. Add or update an eval that would catch the failure next time.

Use `assets/feedback-note-template.md` to capture the issue before editing.
